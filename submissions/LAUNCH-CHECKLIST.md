# Launch Checklist

## Pre-Launch (Before any public posts)

- [ ] Verify all tests pass: `node stress-tests/run-all-experiments.js`
- [ ] Confirm repo is public
- [ ] README leads with evidence
- [ ] Repo description is updated
- [ ] Topics/tags added to repo
- [ ] License file present (MIT)

---

## Week 1: Initial Launch

### Day 1-2: Hacker News
- [ ] Post to Show HN (see `hacker-news-post.md`)
- [ ] Post first comment immediately with details
- [ ] Monitor for questions, respond promptly
- [ ] Be honest about limitations

**Best timing:** Tuesday-Thursday, 9-11am ET

### Day 1-2: Twitter/X
- [ ] Post thread (see `twitter-thread.md`)
- [ ] Tag @AnthropicAI, @ClaudeAI
- [ ] Consider tagging relevant AI/dev influencers
- [ ] Engage with replies

### Day 3-4: Reddit
- [ ] Post to r/MachineLearning (see `reddit-posts.md`)
- [ ] Post to r/programming
- [ ] Post to r/LocalLLaMA
- [ ] Cross-post to r/ExperiencedDevs if relevant

### Day 5-7: Direct Outreach
- [ ] Email Anthropic research (see `email-anthropic.md`)
- [ ] Email 2-3 relevant researchers (see `email-researchers.md`)
- [ ] Open discussion on LangChain GitHub (see `email-langchain-llamaindex.md`)

---

## Week 2-3: Content & Community

### Blog Post
- [ ] Write full blog post (see `blog-post-outline.md`)
- [ ] Create diagrams/charts
- [ ] Publish to Dev.to or Medium
- [ ] Share on Twitter, Reddit, HN

### Community Engagement
- [ ] Respond to GitHub issues
- [ ] Address criticism constructively
- [ ] Incorporate feedback into roadmap
- [ ] Thank contributors

---

## Week 4+: Academic Track

### arXiv Preprint
- [ ] Write full paper (see `arxiv-abstract.md`)
- [ ] Format for arXiv (LaTeX preferred)
- [ ] Submit to cs.SE
- [ ] Update repo with arXiv link

### Conference Submission (if pursuing)
- [ ] Identify target venue (ICSE, FSE, ASE)
- [ ] Check deadline
- [ ] Format per venue requirements
- [ ] Submit

## Tracking Success

### Metrics to Monitor

| Metric | Target | How to Track |
|--------|--------|--------------|
| GitHub stars | 100 in week 1 | GitHub insights |
| HN points | Front page | HN |
| Twitter impressions | 10k | Twitter analytics |
| Repo clones | 50 in week 1 | GitHub insights |
| Issues/PRs | Any engagement | GitHub |
| Email responses | 1-2 | Inbox |

### Qualitative Signals
- [ ] Someone reproduces the results
- [ ] Someone challenges the methodology (good! means they engaged)
- [ ] Someone applies it to their project
- [ ] Framework maintainer expresses interest

## Handling Criticism

**Common critiques and responses:**

| Critique | Response |
|----------|----------|
| "It's just threat modeling" | Yes, applied to AI agents with structural enforcement |
| "Same model shares blind spots" | Agreed, Condition D not yet tested. Structure still helps. |
| "Overhead is too high" | Depends on domain. Skip for prototypes. |
| "Not peer reviewed" | Methodology is public and reproducible. Working on paper. |
| "Simulation, not real agents" | Fair. Production study is on roadmap. |
